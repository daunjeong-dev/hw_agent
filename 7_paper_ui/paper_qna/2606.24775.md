- **summmary**: Memory for large language model (LLM) agents has rapidly evolved from simple retrieval-augmented mechanisms into a data management system that supports persistent information storage, retrieval, update, consolidation, and dynamic lifecycle governance throughout agent execution. Despite this evolution, existing evaluations still benchmark agent memory mainly through end-to-end task success metrics (e.g., F1, BLEU), while treating the underlying system as a monolithic black box. As a result, critical system-level concerns, including operational costs, architectural trade-offs across memory modules, and robustness under dynamic knowledge updates, remain insufficiently explored. In this paper, we present a systematic experimental study of agent memory from a data management perspective.

## Assumption Checker 대화 기록
This paper focuses on the evolution of memory systems for large language model (LLM) agents, analyzing the architectural trade-offs and system-level concerns that existing evaluations often overlook. It proposes an analytical framework that decomposes memory into four core modules: memory representation and storage, extraction, retrieval and routing, and maintenance. This and other findings highlight the importance of tailored memory systems for optimizing agent performance and operational costs, revealing that localized maintenance is more cost-efficient than global strategies, thus paving the way for building effective agent-native memory systems.
